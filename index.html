Gerenciar Exportações de Dados

Como usar
  1. Marque as máquinas desejadas na lista.
  2. Escolha o tipo de exportação: "Linha a linha" ou "Operações".
  3. Se escolher "Operações", selecione ao menos uma operação.
  4. Clique em SALVAR para criar o(s) grupo(s) de exportação.
  5. Nos cartões de resultado:
    • Use Deletar grupo para remover o grupo completo.
    • Clique no botão × ao lado do nome da máquina para removê-la do grupo.
    • Use os chips de adicionar máquina para incluir máquinas ainda disponíveis.
  6. Uma máquina só pode estar em um tipo de exportação por vez. Máquinas já usadas ficam desabilitadas na lista.
Importante: Uma configuração só entra em vigor no início de uma nova operação. Ou seja, máquinas que já estão com operações abertas não terão o formato de envio alterado até que a operação atual seja encerrada e uma nova seja iniciada.
• Linha a linha lista cada ocorrência sem agrupar por operação.
• Operações filtra/agrupa pelos eventos selecionados.
Selecione os eventos operacionais que deseja incluir na exportação.

DDP 356

Documento de Definição de Produto (DDP 356)

Sistema: teepOEE

Assunto: Gerenciamento de Exportações de Dados por Máquina

Empresa: Teep Tecnologia

Participantes

TEEP: Alexandro Trova, Matheus

FACCHINI: Kelvin

Objetivo

Permitir que a Facchini configure exportações de dados individualmente por máquina, definindo o tipo de exportação e, quando aplicável, operações específicas, garantindo flexibilidade e controle sobre o processo.

Funcionalidades propostas
  1. Seleção de máquinas
    • O usuário pode selecionar uma ou mais máquinas para configuração da exportação.
    • Cada máquina só pode estar associada a um único tipo de exportação por vez (linha a linha ou operações).
    • Máquinas já configuradas para um tipo de exportação são indicadas como indisponíveis para o outro tipo, evitando conflitos.
  2. Tipos de exportação
    • Linha a linha: exportar todos os registros individuais da máquina.
    • Operações: exporta dados de eventos específicos da máquina, como:
      • Troca de turno
      • Troca de operador
      • Virada de dia
      • Fechamento de OP
    • Para exportações por operações, é possível selecionar uma ou mais operações.
  3. Agrupamento e gerenciamento
    • Máquinas com o mesmo tipo de exportação e operações selecionadas são agrupadas para visualização e gerenciamento mais fácil.
    • O usuário pode:
      • Remover uma máquina individual do grupo.
      • Remover o grupo completo.
      • Adicionar máquinas disponíveis a grupos existentes.
  4. Validações
    • Pelo menos uma máquina deve ser selecionada para criar a exportação.
    • Um tipo de exportação deve ser escolhido.
    • Para exportações do tipo “Operações”, pelo menos uma operação deve ser selecionada.
    • A lógica impede que uma máquina tenha configurações conflitantes, mantendo consistência nos dados exportados.
  5. Interface de visualização
    • A configuração é apresentada em cascata, mostrando:
    • Tipo de exportação
    • Operações selecionadas (quando aplicável)
    • Máquinas associadas
    • Cada máquina e operação aparece como “chip” visual, com botões para remoção individual.
    • Máquinas disponíveis podem ser adicionadas aos grupos existentes diretamente na visualização.
Benefícios esperados
  • Flexibilidade total na definição de exportações por máquina.
  • Evita conflitos de exportação e duplicidade de dados.
  • Facilidade de gerenciamento visual e manutenção das configurações.
  • Configuração clara e auditável, permitindo rastreabilidade das exportações.
